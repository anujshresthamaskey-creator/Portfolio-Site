Case Study

Secure Login & User Authentication System

Role: Full Stack Developer • Year: 2025 • Platform: Web & Android

Problem

Many login pages look secure but lack essential protection like brute-force prevention, two-factor authentication, and bot validation. Without these, attackers can guess passwords, bypass access, or misuse accounts.

Solution

I developed a secure authentication system using Firebase Authentication, Google reCAPTCHA, OTP-based two-factor authentication, login attempt rate limiting, and password-confirmation before account deletion. This ensures safety while keeping the login process user-friendly.

Email & Password Login
Simple and secure registration and login flow.
Google Sign-In
Quick login using secure OAuth 2.0.
reCAPTCHA Protection
Prevents bots and automated login attempts.
Two-Factor Authentication (OTP)
Verification code sent to email for extra security.
Login Attempt Lockout
Account temporarily locked after 5 failed attempts for 5 minutes.
Secure Account Deletion
User must re-enter password to delete account.
Password Reset
Email recovery link sent via Firebase.

Process — Step-by-Step Usage

How a user interacts with the authentication flow:

1
Create Account
User signs up using email, password & completes reCAPTCHA verification.
2
Login
If login attempts fail 5 times, the user is locked out for 5 minutes.
3
Two-Factor Authentication
User receives a one-time OTP code via email to complete verification.
4
Access Account
User is now securely logged in and session persists until logout.
5
Account Management
To delete the account, user must confirm by entering their password.

Impact

This secure login system prevents most common cyber-attacks such as password guessing, automated bot login attempts, and unauthorized account misuse while keeping the experience smooth.
